## P2: 架构/可读性

### [P2-1] 线程安全声称与实现不一致 🟡

位置: src/frame_stats.rs:31 注释 + 结构体定义

```rust
/// 帧率采集器。非线程安全,由渲染循环单线程持有。
#[derive(Debug)]
pub struct FrameStats {
    last_present: Option<Instant>,
    intervals: Vec<Duration>,
}
```

问题:注释说"非线程安全",但结构体自动 derive `Send + Sync`(因为 Vec<Duration> 和 Instant 都是)。多线程并发调用 record_present 会 race(不会 panic,只是逻辑错乱)。

推荐修法(可选):
- A:删除注释(因为实际是 Send/Sync)
- B:加 `PhantomData<*const ()>` 编译期强制移除 Send

判定:🟡 低优先级,单线程使用完全安全
