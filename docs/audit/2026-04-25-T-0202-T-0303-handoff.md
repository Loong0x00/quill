## 1. 类型隔离范式 (T-0302/T-0303 共建)

**约束**: alacritty 类型 (`Point`, `CursorShape`, 未来 `Color`) 彻底锁在 `src/term/mod.rs` 内, 公共 API 零 alacritty 类型。`grep -nE 'pub use alacritty|pub.*alacritty::' src/` 应零命中, 任何违反是 P0 阻塞。

**为什么**: alacritty 0.x 主版本的内部类型不稳定 (例 Point 是 `{Line(i32), Column(usize)}` 不对称, scrollback 用负 i32 line)。渲染层只关心 viewport 0..rows × 0..cols 的 usize 坐标, 暴露上游类型让下游背包袱; 未来换 VT 库 (kitty / termwiz) 会 cascade 改下游所有调用点。

**实施模式** (T-0302 fixup2 + T-0303 共固化):
- quill 自定义 `pub struct CellPos { col: usize, line: usize }` / `pub enum CursorShape { ... }`
- 转换走**模块私有 inherent fn** `fn from_alacritty(p: AlacPoint) -> Self` — **不**用 `impl From` trait (trait impl 会让下游 `p.into()` 反向构造, 漏出类型路径)
- enum 转换用 **exhaustive match 无 `_ =>`**, 编译期 catches alacritty 加新 variant
- 数值转换加 **saturating cast** (`.max(0) as usize`) release 也防御负值漏入

**扩展 hook**: T-0305 加 `pub struct Color { r,g,b,a: u8 }` (不 re-export alacritty Color)。T-0304 scrollback 推荐独立 `pub struct ScrollbackPos` 类型 (不扩 CellPos enum), 同款 from_alacritty 私有 fn。

## 2. POSIX poll(2) 语义: 必须 `.intersects(IN | HUP | ERR)`

**约束**: 检查 fd 可读性时, 不能只 `revents().contains(IN)` — POLLHUP 和 POLLERR 在 Linux 上是 **output-only revents**, 即使没在 events 里 request 也会自动出现, 而且**单独发, 无 IN 伴随**。

**为什么**: T-0205 修的关键 bug — Linux PTY master 在 slave 关闭且 master 缓冲已排空时**仅返 POLLHUP 无 POLLIN**。旧 `contains(IN)` 漏过 → callback 不跑 → EOF 永远测不到 → quill 不退出 (pkill bash 后僵死)。

**正确写法**: `revents().intersects(PollFlags::IN | PollFlags::HUP | PollFlags::ERR)`。实际读字节由 read 返回值精细分类 (T-0205 的 `pty_readable_action` 纯函数: `Ok(0)/EIO → RequestExit`, `EAGAIN → ReturnContinue`, etc)。

**扩展 hook**: 项目 rustix 直接 poll 已被 T-0108 calloop 统一替换, 暂无新使用点。若 Phase 5+ fcitx5 接 IPC fd 用 poll, 复用此规则。

## 3. calloop 三源统一架构 (T-0108)

**架构**: `EventLoop<LoopData>` 调度 wayland_fd / signalfd / pty_fd 三条 source。`LoopData = { event_queue, state, term, loop_signal }` 聚合所有运行时对象, callback 用 split borrow `let LoopData { state, term, loop_signal, .. } = &mut *data` 同时拿 `&mut state.pty + &mut term + &loop_signal`。

**为什么**: T-0104 时代手写 `rustix::event::poll` + signal-hook self-pipe + Arc<AtomicBool> 的 workaround, 三套机制并存 (TD-005 双 poll + TD-006 signal handler 纳秒竞态 + TD-001 Core Data 升级悬挂) 不可持续。T-0108 一次清掉三条 TD, 走 calloop idiomatic 单 EventLoop + signalfd + LoopSignal::stop()。

**关键观察**:
- Term 放 LoopData 不放 State — wayland Dispatch trait 不需要 term, State 干净专注 wayland callback
- 退出统一经 `loop_signal.stop()`, 四条触发路径 (xdg close / SIGINT / SIGTERM / pty EOF) 同一出口
- signalfd 比 self-pipe + Arc<AtomicBool> race-free (TD-006 彻底消除)

**扩展 hook**: T-0306 Wayland resize → pty.resize 链路在 wayland source callback 里加, 不冲突 (`pty.resize(&self)` 与 pty_read_tick 的 `&mut pty` 不会同时持)。T-0501+ fcitx5 IPC fd 加第 4 条 source。

## 4. PtyHandle 五方法成熟度 (Phase 2 完整闭环)

`spawn_shell` / `spawn_program` / `raw_fd` / `read` / `resize` / `try_wait` 全实装。不变式:
- **INV-008**: 字段顺序 reader → master → child → master_fd, drop 序保证 SIGHUP 在 child reap 之前
- **INV-009**: master fd `O_NONBLOCK` 在 spawn 时 fcntl 一次设好, 后续禁止重设 F_SETFL (会覆盖其它 flag)
- **#[allow(dead_code)] 已移除** (T-0205): 五方法都被消费

**关键经验**: `child.try_wait` 不阻塞, `Ok(None)` race 接受不重试 (zombie 由 init adopt 兜底)。callback 错误处理三档: `Ok(0)/EIO/未知 errno → RequestExit + LoopSignal::stop`, `WouldBlock → ReturnContinue`, `EINTR → 重试`。

**扩展 hook**: T-0306 `resize(&self)` 已就绪, 直接从 wayland configure 回调里 `state.pty.as_ref().map(|p| p.resize(cols, rows))`。Phase 4+ 字形渲染时 cell pixel size 测量好后再调。
