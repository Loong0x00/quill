### [P0-2] CursorShape 5 variants 1:1 映射 + exhaustive match ✅
```rust
fn from_alacritty(s: alacritty_terminal::vte::ansi::CursorShape) -> Self {
    use alacritty_terminal::vte::ansi::CursorShape as Up;
    match s {
        Up::Block => CursorShape::Block,
        Up::Underline => CursorShape::Underline,
        Up::Beam => CursorShape::Beam,
        Up::HollowBlock => CursorShape::HollowBlock,
        Up::Hidden => CursorShape::Hidden,
    }
}
```
- 无 `_ =>` 通配 → 编译期 catches alacritty 加新 variant 的回归 ✅
- HollowBlock **不**折叠到 Block — **正确决策**: HollowBlock 是 blur 时显示形状, fold 进 Block 会让渲染层无法区分 focus / blur 状态, Phase 4+ 焦点 UX 时会用到这个区分
- Hidden 单独 variant 不折叠到 cursor_visible() 关 — **正确, 见 P0-3**

### [P0-3] cursor_shape / cursor_visible 正交设计独立复判 ✅ (比 alacritty 内部更干净)
独立分析:
- alacritty 的 `CursorRenderingData::new` 在 SHOW_CURSOR mode 关时会**主动**返 Hidden, 综合两个状态成一个值
- quill 拒绝这种综合, 显式拆开: `cursor_visible() -> bool` 读 mode 位; `cursor_shape() -> CursorShape` 读 shape 字段
- 渲染层组合 `if visible { draw(shape) }` —— 语义清晰, 调用方知道两层独立含义
- docstring 明确 "Hidden 与 cursor_visible() == false **正交, 两个都得查**" 把约束记录下来

这是**比上游设计更干净**的 API choice, 加分。

### [P2-2] 渲染层调用链一致性 ✅
T-0303 后 T-0305 渲染调用链:
```rust
let (cols, rows) = term.dimensions();          // T-0302
for cell in term.cells_iter() {                 // T-0302
    draw_cell_at_pos(cell.pos, cell.c);         // CellPos
}
if term.cursor_visible() {                      // T-0302
    let pos = term.cursor_pos();                // T-0303 ← 与 cell.pos 同类型
    draw_cursor(pos, term.cursor_shape());      // T-0303
}
```
**整条链类型一致** (CellPos / CursorShape / bool / dimensions tuple), 渲染代码不会出现"光标位置类型 vs cell 位置类型不兼容"的怪。 ✅

### [P2-3] cursor_shape vs cursor_visible 正交 (再强调)
docstring 给 T-0305 渲染层留了**显式组合契约**:
```
if t.cursor_visible() { draw_cursor(t.cursor_shape()) }
```
两层各自单调语义, 调用方思考粒度小。

### [P2-4] 不暴露 blinking 信息 ✅

## 给后续 ticket 的前置信息

### 给 T-0304 (scrollback)
- CellPos 现在 `{ col: usize, line: usize }` 严格 viewport 坐标
- T-0304 scrollback 历史行需要负 line 概念时, 选项:
  1. 扩 CellPos 成 enum `{ Viewport(...), Scrollback(usize) }` — 破坏现有 cells_iter / cursor_pos 调用方
  2. 加独立 `ScrollbackPos` 类型 — 不破坏现有 API, 独立通路
- 推 (2)。复用 from_alacritty 模式做私有 inherent fn

### 给 T-0305 (色块渲染)
- API 完备, 渲染调用链类型一致, 直接消费即可
- CellRef 暂无 fg/bg, T-0305 ticket 应要求加 `Color` (quill 自定义 struct, 不 re-export alacritty Color, 同样套路)
- cursor_shape 五 variants 中 HollowBlock 最特殊 (空心), T-0305 渲染时要决定 "色块时代" 怎么画 HollowBlock — 可能简化 fold 进 Block + 加注释 "Phase 4 字形渲染再细化", 也可能直接画矩形外框。不在 T-0303 scope。

### 给 T-0306 (resize)
- dimensions() 已实时读 alacritty 内部 — resize 后无需手动同步
- cursor_pos / cursor_shape 不受 resize 影响 (光标本身随 grid 内部状态走)
