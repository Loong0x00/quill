### [P0-2] DecorationLogDecision enum + decoration_log_decision fn 走 conventions §3 抽决策套路 ✅

writer 把"compositor reply 装饰模式 → log 决策"抽成纯 fn `decoration_log_decision(DecorationMode) -> DecorationLogDecision`, 跟 conventions §3 sample (`pty_readable_action` / `verdict_for_scale` / `cells_from_surface_px` / T-0302 `from_alacritty`) 一脉相承:

```rust
#[derive(Debug, Clone, Copy, PartialEq, Eq)]
pub(crate) enum DecorationLogDecision {
    ServerSideAccepted,
    ClientSideFallback,
}

pub(crate) fn decoration_log_decision(mode: DecorationMode) -> DecorationLogDecision {
    match mode {
        DecorationMode::Server => DecorationLogDecision::ServerSideAccepted,
        DecorationMode::Client => DecorationLogDecision::ClientSideFallback,
    }
}
```

- 抽 enum 而非 bool: 未来扩 CSD-self-paint / hybrid 装饰不破 enum ABI
- 抽纯 fn: 单测覆盖两种已知 variant → 两种 log 决策, 不依赖真 Wayland
- `pub(crate)`: 模块外不可见, INV-010 类型隔离 (DecorationMode 是 sctk re-export, 不流出本 module 边界)

`WindowHandler::configure` 调用方按 enum 分派 `tracing::info!` (Server) / `tracing::warn!` (Client), 副作用与决策分离干净。

### [P0-4] WindowCore::decoration_logged 一次性 log 标记语义正确 ✅

字段位于 `WindowCore` (T-0107 抽出来的纯逻辑核心), 跟 `first_configure` / `exit` / `resize_dirty` 同位:

```rust
pub struct WindowCore {
    pub width: u32,
    pub height: u32,
    pub first_configure: bool,
    pub exit: bool,
    pub resize_dirty: bool,
    pub decoration_logged: bool,  // T-0503 加
}
```

`WindowHandler::configure` 调用路径:

```rust
if !self.core.decoration_logged {
    match decoration_log_decision(configure.decoration_mode) {
        DecorationLogDecision::ServerSideAccepted => tracing::info!(...),
        DecorationLogDecision::ClientSideFallback => tracing::warn!(...),
    }
    self.core.decoration_logged = true;
}
```

- 一次性: 第一次 configure 后置 true, 后续 configure (focus / resize / state 变化每次都 fire) 不重复 log → trace 不污染
- 装饰模式生命周期内不变 (协议规约): 一次 log 决策足够, 不漏关键诊断信号
- 字段标 `pub` 跟其它 `WindowCore` 字段一致, headless 测试可独立读写 (tests/xdg_decoration.rs 第 3 个测试就显式锁这条可见性)

INV-007 ("WindowCore state machine 是纯逻辑, 无副作用") 维持: 字段只在 `WindowHandler::configure` (Wayland 真回调) 改, `handle_event` 不碰 (装饰协商不通过状态机, 直接读 configure 参数).
