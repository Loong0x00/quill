## 3. 3 偏离 accept/reject

| # | 偏离 | accept/reject | 理由 |
|---|------|---------------|------|
| 1 | `tick_repeat()` 签名去 `now: Instant` 入参 | ✅ accept | 派单 In #A 写 `tick_repeat(state, now: Instant) -> Option<Vec<u8>>`, 但 calloop `Timer::from_duration` + `TimeoutAction::ToDuration` 已让 calloop 内核做精确调度, callback 不需要重算 deadline。如果 keep `now`, 就是 dead arg + 不必要 import std::time::Instant 进 keyboard.rs (该模块仅用 xkb 类型, 不引时间)。简化 API 是对的, 注释 (line 186-189) 已显式说明派单 vs 实装差异。 |
| 2 | 集成测试用真 calloop Timer + wall-clock dispatch slice (不 mock raw timerfd) | ✅ accept | 派单 In #B 明文 "实际推荐 calloop::timer::Timer, 不用 raw timerfd" + 派单 In #D 第 3 条 "mock timerfd, fire 模拟" 描述, 但**派单允许走 calloop Timer** (硬约束: "用 calloop::timer::Timer")。集成测试用真 Timer + wall-clock dispatch 是与派单一致的做法; 真 mock raw timerfd 不仅复杂 (重写 calloop 内部 fd 读) 还违背派单 hint。文件头注释 (line 11-13) 明示 "calloop Timer 不 mock timerfd 系统调用本身"。 |
| 3 | `KeyboardAction::WriteToPty` + `#[allow(dead_code)]` 保留 | ✅ accept | T-0603 起所有 utf8 非空 Pressed 都走 StartRepeat 路径 (含立即写一次的 bytes), WriteToPty 现在没人构造但 Dispatch<WlKeyboard> match 仍需该 variant (无 `_ =>` 兜底 = INV-010 enum 防御编译期 catch 上游 / 自有 enum 加新 variant)。删 variant 就破 enum 防御性。`#[allow(dead_code)]` 显式标注 + 注释 (line 226-230) 说明保留理由 + 未来 forward-compat (F1-F12 escape sequence 决定不连发, 或 IME compose 中间态需回显但不 repeat) — 对的决策。**比"删除"严更好**。 |

**3 偏离全 accept** (1 个是 API 简化 / 1 个是派单 hint 优先 / 1 个是 enum 防御性保留, 全在派单 In scope 内的精化或硬规范坚持)。
