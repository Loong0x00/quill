### [P0-4] 越界行为 ✅ (clamp 不 panic, 显式分支兜底)
独立验证 `to_alacritty` 实装:
```rust
fn to_alacritty(self, history_size: usize) -> Line {
    if history_size == 0 {
        return Line(0);                                  // 兜底分支 1
    }
    let history = history_size as i32;
    let row = (self.row as i32).min(history - 1);        // clamp 越界
    Line(row - history)                                   // row=0 → -history (最旧)
                                                          // row=size-1 → -1 (最新)
}
```
三档行为:
- **正常**: row=0 → Line(-history) 最旧; row=history-1 → Line(-1) 最新
- **越界 row >= history_size**: clamp 到 Line(-1), 落在最新滚出行 (而非 panic)
- **history_size == 0**: 显式分支落 Line(0) (viewport 第 0 行, 无历史可索引)

测试 `scrollbackpos_to_alacritty_boundaries` 全 5 case 覆盖:
- `(row=0, history=10) → Line(-10)` ✅
- `(row=9, history=10) → Line(-1)` ✅
- `(row=999, history=10) → Line(-1)` clamp ✅
- `(row=0, history=0) → Line(0)` 兜底 ✅
- `(row=5, history=0) → Line(0)` 兜底 ✅

`history_size == 0` 时 line_text/cells_iter 会读 viewport 第 0 行内容(非 panic
也非空字符串), 是显式 trade-off: 调用方应先 `scrollback_size()` 校验。docstring
明确该约束。语义稍 surprising 但已显式记录, 不阻塞 (见 P3-3)。

### [P2-2] 渲染层调用链类型一致性 ✅
T-0304 后 T-0305 渲染调用链 (含 scroll-up UI 设想):
```rust
// viewport 当前帧
let (cols, rows) = term.dimensions();
for cell in term.cells_iter() { draw_cell(cell.pos, cell.c); }
if term.cursor_visible() { draw_cursor(term.cursor_pos(), term.cursor_shape()); }

// scroll-up: 显示历史行 N
let pos = ScrollbackPos { row: 0 };
for cell in term.scrollback_cells_iter(pos) {
    draw_history_cell_at(scroll_screen_y, cell.pos.col, cell.c);
    //                    ^^^^^^^^^^^^^^^ 由 ScrollbackPos 单独承载, 不用 cell.pos.line
}
```
viewport / scrollback 两条通路类型分离 (CellPos / ScrollbackPos), 渲染层显式
区分两类位置, **不混用**。CellRef 类型复用 → 绘制函数可同一份。

### [P2-3] 三个 pub 方法 API 内聚 ✅
- `scrollback_size()` — 状态查询, 与 `dimensions()` 同档
- `scrollback_line_text(pos)` — 字符串读, 与 `line_text(line)` 对应
- `scrollback_cells_iter(pos)` — cell 迭代, 与 `cells_iter()` 对应

完整对偶 viewport API 三件套。CellRef 类型复用而非另起 `ScrollbackCellRef`,
减少类型表面积, 但代价是 line 字段语义占位 (见 P3-2)。

### [P2-4] row 方向选择 (row=0 最旧) ✅
docstring 明确说明 quill 对外 row 方向**反向**于 alacritty 内部 Line:
- quill: row=0 最旧 (scrollback 顶端) → row=size-1 最新滚出
- alacritty: Line(-1) 最新 → Line(-history) 最旧

`to_alacritty` 做反向映射, 下游不感知。**好处**: scroll-up UI "往上滚 N 行 = row 减少",
与用户认知 (上 = 旧) 自然一致。**潜在踩坑**: 调试 alacritty grid 时与 quill row
方向相反, 但由 docstring 显式记录, 接班开发可读。

## P3 建议 (不阻塞)

### [P3-1] CellRef line=0 占位是否值得新类型 (T-0305 时再评)
当前 `scrollback_cells_iter` 复用 CellRef → `pos.line=0` 占位, 真实位置由
ScrollbackPos 承载。T-0305 渲染层若发现"经常忘了 ScrollbackPos 而误用 cell.pos.line",
可以加 `ScrollbackCellRef { col: usize, c: char }` 单独类型 (无 line 字段, 类型层面
防误用)。

但当前 trade-off 合理:
- 复用 CellRef 减少类型数, 渲染绘制函数可共用 `(col, c)` 部分
- docstring + 测试断言 `pos.line == 0` 已锁占位
- 加新类型是后续 ticket 的事, 现状不阻塞

### [P3-2] history_size==0 时 line_text/cells_iter 兜底语义
当 `history_size == 0` 时, `scrollback_line_text` 走 to_alacritty 兜底返
`Line(0)` → 实际读 viewport 第 0 行。语义稍 surprising (调用方期望 "无历史" 应返
空 / Option::None), 但 docstring 明确"调用方应先 scrollback_size() 校验", 是显式
契约。

可选改进 (后续 ticket):
- 改 API 为 `scrollback_line_text(pos) -> Option<String>` (history_size==0 或 row 越界返 None)
- 这与 `line_text(line)` 不一致 (后者也不返 Option, 越界 panic), 改一致性需要整体重新审视

不阻塞当前合并。

### [P3-3] 写码自加测试 `scrollbackpos_to_alacritty_boundaries` 评价
派单要求 4 个测试, 写码加了 5 个 (多 `to_alacritty` 私有 fn 边界测试)。**加分**:
- 私有 fn 直测覆盖 3 档行为 (正常 / 越界 clamp / history=0 兜底), 比黑盒经过
  pub API 间接测试更严, 防回归边界 case (T-0302 同款风格: `cellpos_from_alacritty_viewport_and_negative_line`)
- 5 case 紧凑 (单测试体内 5 assert), 不冗
- 同文件 tests mod 走私有 fn 是 conventions.md §3 鼓励的"复杂决策抽纯函数 + 单测"
  套路

### [P3-4] scrollback 上限测试缺失
派单 / 写码均未覆盖 `scrolling_history=10000` 上限达到后旧行被丢弃的行为
(scrollback_size 不会超过 10000)。这是 alacritty 内部 ring-buffer 行为,
归属于 alacritty 测试责任, quill 不必重复覆盖。但若未来 T-0306 resize 后
history capacity 变化, 可加测试锁住"capacity 满时 row=0 被替换"。当前不阻塞。
