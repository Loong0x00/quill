### P3 (nice-to-have)

**[P3-1] CursorInfo 字段加 `#[non_exhaustive]` 防下游 struct literal 锁死**

`CursorInfo { col, line, visible, style, color }` 5 字段 pub, 当前 5 字段 = 当前需求。未来若加新字段 (例: blink_phase / focus_state), 下游 struct literal `CursorInfo { col, line, visible, style, color }` 必破 (semver-major)。加 `#[non_exhaustive]` 强制下游用 builder / `..Default::default()` 构造, 加字段不破 API。

**不阻塞** (当前 quill 自家用, 无下游); 但 `#[non_exhaustive]` 是 Rust API 演进 idiom 成本极低收益高, Phase 6+ 加 blink 时一起改即可。
