### B. src/wl/pointer.rs 加 cursor shape 状态 PASS

- `CursorShape` enum 10 variant 中实际只 6 (Default/Text + 4 resize), 派单文字 "10 variants" 是 ADR 草拟时口径, 实装为最小 6 个 — 4 resize variant 加 `#[allow(dead_code)]` + 注释指向 T-0701 合后填 ResizeEdge 分支 (派单选项 B 路径).
- `cursor_shape_for(HoverRegion) -> CursorShape` 翻译表 fn (5 case exhaustive match, 当前 ResizeEdge variant 不存在所以 None/TitleBar/Button → Default, TextArea → Text)
- `wp_shape_for(CursorShape) -> WpShape` 模块私有 inherent fn 转换 (`pub(crate)` 给 window.rs Dispatch 段调用), 不暴露 WpShape 到下游
- `PointerState` 加 3 字段 (`last_enter_serial` / `current_cursor_shape` / `pending_cursor_set`), `pending_cursor_set: Option<(u32, CursorShape)>` 走 take 模式与 T-0602 `pending_scroll_lines` / T-0603 `pending_repeat` 同套路
- `apply_enter` signature 加 serial (协议要求): 强制 emit 一次 (即便 cursor 形状未变 — 协议: enter 后 client 必须 set 一次 cursor, 否则 cursor 行为 unspecified, 实测 mutter 显示空白). `apply_motion` 跨 hover 时若 cursor 形状变才 emit (n:1 映射优化, 同 cursor 不重发减压 compositor).
- `apply_leave` 清空 `pending_cursor_set` (兜底 Enter 后立刻 Leave 的 race).

### C. src/wl/window.rs Dispatch 接 SetCursor PASS

- `run_window` 启动期 `globals.bind(&qh, 1..=1, ())` 锁 v1 (上游 v2 加 dnd_ask + all_resize 不需要), Ok → Some + info log, Err → None + warn log
- `State` 加两字段 `cursor_shape_manager` / `cursor_shape_device` 位于 `pointer_seat` 之后 / `is_maximized` 之前 (在 pty 之前) — INV-001 链条 (renderer→window→conn) 完整, 这两个 wayland Proxy 与 keyboard/pointer 同性质 wayland safe wrapper, drop 顺序无 UB 风险

- `SeatHandler::remove_capability` (Pointer) 时 `cursor_shape_device.take()` (协议明示: pointer capability 移除时 device 自动 inert), `remove_seat` 同理 take
- `Dispatch<WpCursorShapeManagerV1, ()>` + `Dispatch<WpCursorShapeDeviceV1, ()>` 两个 防御性 impl (协议双方都零 event, 防 trait bound 过不去)

## 6. 三偏离审视 (writer 主动告知)

### 偏离 1: `apply_enter` signature 加 `serial: u32` accept

**writer**: 协议要求 set_shape 用 enter serial (motion event 自身无 serial 字段), apply_enter 必须接 serial 记入 `last_enter_serial`. 12 个老 apply_* 测试更新加 `serial=1` 占位调用, 行为零变化 (其它测试不读 serial 字段).

**审码评注**: 这是协议合同的直接体现 — wp_cursor_shape_device_v1.set_shape 文档原话 "The serial parameter must match the latest wl_pointer.enter ... serial. Otherwise the request will be ignored." motion 事件自身无 serial, 必须复用最近 enter serial. signature 改 + state 字段加 + 测试占位 是闭环最小变更. 验证: `axis_*` / `hit_test_*` 等 8 个不涉 enter 的测试零变化, 仅 `enter_*` / `apply_button` 等显式 enter 路径加占位 serial=1, 不影响断言. **accept**.

### 偏离 2: `PointerState` 加 3 字段 (`last_enter_serial` / `current_cursor_shape` / `pending_cursor_set`) accept

**writer**: pending_cursor_set 是 sink 模式跟 T-0603 pending_repeat / T-0602 pending_scroll_lines 同套路, 单 buffer 设计, take 后置 None.

**审码评注**: 这套 take 模式是 conventions 沉淀的标准做法 — Dispatch 段在主 PointerAction 处理后取 pending 副作用下发, 一帧多次填覆盖前次 (不积压, 取最新), Leave 兜底清防 race. `current_cursor_shape` 用作 n:1 映射优化 (titlebar / button 都给 Default cursor, 跨这两个 region 不重发 set_shape, 减压 compositor + 防 cursor 闪烁), 单测 `motion_across_equivalent_cursor_regions_does_not_reemit` 锁住. `last_enter_serial` 是协议刚需. 三字段都对应严格的协议/性能需求, 不是 over-engineering. **accept**.

### 偏离 3: 4 resize CursorShape variant 加 `#[allow(dead_code)]` accept

**writer**: T-0701 ResizeEdge enum 合并前 cursor_shape_for 无 ResizeEdge 分支 → 不返此 variant. Lead 合并 T-0701 后手加 `cursor_shape_for` ResizeEdge 分支即激活 (派单选项 B).

**审码评注**: 派单原文允许选项 B (先实装 T-0703 不含 ResizeEdge, Lead 合并 T-0701 时手加 ResizeEdge mapping). 4 variant 各自带详细注释引用 T-0701 合后填的具体 ResizeEdge case (Top/Bottom→NsResize, Left/Right→EwResize, TopLeft/BottomRight→NwseResize, TopRight/BottomLeft→NeswResize), `wp_shape_for` 已实装全 6 variant 映射 (含 4 resize, 单测锁住). T-0701 合后只需:
1. cursor_shape_for 加 `HoverRegion::ResizeEdge(edge) => match edge { ... }` 分支 (~10 行)
2. 移除 4 个 `#[allow(dead_code)]`
即激活. 与 ADR 0008 实装验证段一致. **accept**.

## 7. 代码 micro-review (亮点 + 风险)

### 亮点 (无需 action)

1. **协议事件分派表注释** (`handle_pointer_event` 文档头) 列全 11 个 wl_pointer event variant + 处理状态, 便于后续 audit 比对协议覆盖度. T-0703 在原 T-0504 表上加了 Enter serial 用途说明 + 协议陷阱段加 enter vs button serial 区分.

2. **Enter 强制 emit 但同 cursor 不在 motion 重发** (`apply_enter` vs `apply_motion`) — 协议要求 + 性能优化双兼顾. 单测 `enter_always_emits_pending_cursor_set_with_enter_serial` + `motion_across_equivalent_cursor_regions_does_not_reemit` 双锁.

3. **Leave 清 pending_cursor_set 兜底 race** (`apply_leave`) — Enter 后立刻 Leave 时 pending 未消费, 鼠标已离开 surface 再 set_shape 也 stale. 防御性清理 + 单测 `leave_clears_pending_cursor_set` 锁住.

4. **set_shape 路径放 Dispatch<WlPointer> 闭包前段** (在主 PointerAction match 之前), 注释 "Enter event 也走 set_cursor 路径" 说明顺序考量 (Enter 也是 wl_pointer event, 要确保 take_pending 先于 PointerAction::HoverChange 处理 — 实际两者顺序无 race, 但单点消费更清晰).

5. **退化路径 trace 不 warn** (cursor_shape_device=None 时), 注释 "防刷屏" — set_shape 调用频率与 motion 同, warn 会刷飞 log, trace 是合适级别. 与 conventions §6 "trace 是 out-of-band" 一致.

### 风险 (无 action, 仅供后续追踪)

1. **wp_cursor_shape_v1 v1→v2 升级** (上游加 dnd_ask + all_resize cursor): 当前锁 `1..=1`, 升级时需手改 version range + 评估是否需要新 CursorShape variant. ADR 0008 已注明.

2. **手测 deliverable** 需 Lead 在 GNOME Wayland session 跑 `cargo run --release` 实测 cursor 切换 (titlebar=Default, textarea=I-beam). 单测覆盖 client 侧逻辑闭环, 但 compositor 侧渲染需视觉 verify. 4 边 4 角 resize cursor 等 T-0701 合并后再测.
