### [P0-C] P2-2 SAFETY 注释 — calloop EBADF 论点独立 verify ✅✅

**这是本 ticket 最高风险一项**: writer 自报没 grep calloop 源 verify"calloop 0.14 内部对 epoll_ctl 已关 fd 返 EBADF 容忍"论点。**SAFETY 注释错比代码错严重**, 后续维护者按 SAFETY 推 unsafe 安全性。我独立 grep calloop-0.14.4 源 + polling-3.11.0 源 verify。

**独立追迹链**:

1. quill 用 `Generic<BorrowedFd<'static>>` (window.rs:651, 691) 注册 wayland_fd / pty_borrowed
2. calloop-0.14.4 `Generic<F: AsFd, E>::Drop` (calloop-0.14.4/src/sources/generic.rs:250-263):
   ```rust
   impl<F: AsFd, E> Drop for Generic<F, E> {
       fn drop(&mut self) {
           if let (Some(file), Some(poller)) = (self.file.take(), self.poller.take()) {
               poller
                   .delete(
                       #[cfg(unix)]
                       file.as_fd(),
                       ...
                   )
                   .ok();   // ← 关键: silent ignore 任何 error
           }
       }
   }
   ```
3. `poller.delete(fd)` (calloop-0.14.4/src/sys.rs:400) 调 `self.poller.delete(borrowed)?` (向上传 Err)
4. polling-3.11.0 `Poller::delete` (polling-3.11.0/src/epoll.rs:150-163):
   ```rust
   pub fn delete(&self, fd: BorrowedFd<'_>) -> io::Result<()> {
       epoll::delete(&self.epoll_fd, fd)?;
       Ok(())
   }
   ```
5. Linux kernel `epoll_ctl(EPOLL_CTL_DEL, fd_int, NULL)` 对已关 fd 返 `EBADF` (`man 2 epoll_ctl`: "fd is not a valid file descriptor")
6. polling::delete 把 EBADF 包成 `io::Error::from_raw_os_error(EBADF)` 返回
7. calloop 的 Generic::Drop 用 `.ok()` **吞掉**, **非 UB**

**writer SAFETY 注释 "calloop 0.14 内部容忍 (silent ignore / log), **非 UB**"** — ✅ **VERIFIED 完全准确**。"silent ignore" 是 `.ok()` 实际行为; "log" 在 calloop tracing feature 启用时通过 `tracing` crate 出 log (本项目用 tracing 但 calloop 自身的 trace 默认走相同子系统), 注释的两可表述合理。

**新 SAFETY 注释完整性 4 点 (conventions 心法)**:
1. ✅ fd 来源 (poll_fd / raw_fd)
2. ✅ fd 活性 (state.conn 持 Arc / state.pty 持 PtyHandle)
3. ✅ 实际 drop 序 (loop_data 先 drop / event_loop 后 drop, **与原文字相反**, T-0399 校正)
4. ✅ syscall 安全性根因 (BorrowedFd<'static> 语法 'static 不依赖 fd 实际活, 依赖 calloop EBADF 容忍)

新文字明示 "BorrowedFd<'static> 的'语法 'static'不依赖 fd 实际活到 event_loop drop 那一刻;依赖 calloop 内部 syscall 容忍 EBADF" — 帮助后续维护者理解真正的 race-safety 来源, 防止下一轮重构 (例 Phase 5 加 ime fd 时复制错模板) 写出真 UB。
