### 3.2 titlebar_title_x_start (居中算法)

`src/wl/render.rs::titlebar_title_x_start` (line 506-518) 是 free fn (与
`append_quad_px` / `append_preedit_underline_to_cell_bytes` 同决策, 让 draw_frame
+ render_headless 共用纯逻辑):

```rust
fn titlebar_title_x_start(surface_w: f32, title_advance: f32) -> f32 {
    if title_advance >= surface_w {
        0.0
    } else {
        (surface_w - title_advance) / 2.0
    }
}
```

- 派单关键提示原文 `x = (surface_w / 2) - (title_advance / 2)` 等价于 `(sw - adv) / 2` ✅
- title 比 surface 宽时 (`title_advance > surface_w`) 退化为 0 防 NDC pos.x < -1 跑负
- `>=` 而非 `>` 包含 `adv == sw` 边界 (字铺满 surface 时居中起点 0, 视觉等价)
- **3 单测覆盖**: fits / equals_width / overflow

### 3.4 draw_frame self.title.clone() (borrow checker 路径)

`src/wl/render.rs::draw_frame` (line 1542-1562):

```rust
let title_color = self.color_for_vertex(BUTTON_ICON);
let title_for_render = self.title.clone();   // ← 微秒级 clone
self.append_titlebar_title_glyphs(
    text_system,
    &mut glyph_vertex_bytes,
    &title_for_render,
    surface_w,
    surface_h,
    titlebar_y_offset_px,
    title_color,
);
```

**clone 必要性** (writer doc 已声明):
- `append_titlebar_title_glyphs` 入参 `&mut self` (要 &mut self.glyph_atlas /
  &self.queue / &mut self.device 写 atlas) + `title: &str` (要 self.title)
- borrow checker 不允许同时存在 `&mut self.glyph_atlas` (来自 &mut self) 与
  `&self.title` (来自 &self) — 必须先 clone out
- title 长度 ≤ 32 chars 实战 (Phase 7+ cwd 100 chars 上限), ASCII String::clone
  ~10 ns, 远低于 wgpu queue.write_texture (~µs) — clone 路径 KISS 接受
- **替代路径**: 改 `append_titlebar_title_glyphs` 入参为 `&str` 不 borrow self,
  把 self.append → 模块级 free fn 走 raw &mut device/queue/atlas — 改动量大,
  与现有 `append_preedit_glyphs` 同套路 (也 borrow &mut self), 偏离 KISS 反而
  破坏一致性. clone 路径正确 ✅
