## P0 全部通过

### [P0-1] calloop::signals 替代 self-pipe ✅
- `Signals::new(&[SIGINT, SIGTERM])` 走 signalfd (Linux), 信号进 calloop 统一 poll
- 不捕 SIGHUP (session manager 处理)
- callback → `loop_signal.stop()` 让 EventLoop 当前 dispatch 结束即返回

### [P0-2] TD-006 纳秒级竞态窗口彻底消除 ✅
ADR 0003 原 race 路径 "signal handler → Arc store + pipe write → poll 醒" 替换为 "signal → kernel signalfd → poll 立刻见 fd ready → callback stop()"。用户空间无 race gap。

### [P0-3] TD-005 双 poll 消除 ✅
`grep 'rustix::event::poll' src/` 零命中, 唯一 calloop EventLoop 调度三源。

### [P0-4] TD-001 Core Data 升级 (方案 A LoopData) ✅
独立复判 Term 放 LoopData 不放 State **合理**:
- State 已满 CompositorHandler/OutputHandler/WindowHandler + 多个 delegate_* 宏, Term 进 State 污染 Dispatch mental model
- LoopData 是 calloop 数据层, 装 event_queue + state + term + loop_signal 是天然聚合
- T-0305 反悔风险 = 小摩擦点 (不是阻塞)

### [P0-5] LoopData split borrow 合法 ✅
`let LoopData { state, term, loop_signal, .. } = &mut *data` 字段级 split + `state.pty.as_mut()` 二级借用, Rust 2021 NLL 稳定支持。

### [P0-6] wayland_fd 进 calloop 四步逻辑 ✅
drive_wayland: prepare_read / read / dispatch_pending / flush, EINTR 由 calloop 自动 retry, signalfd 在 signal 到达后立刻 ready, retry 的 poll 立即返回 → signal callback 跑 stop()。无漏网。

### [P0-7] stop() 到 run() 返回 <1s ✅
手测四退出路径 (SIGINT / SIGTERM / pkill bash / xdg close) 均 <1s 退码 0。

### [P0-8] EventLoop drop 顺序维持 INV ✅
event_loop → loop_data → event_queue → state → term → loop_signal, state drop 触发 INV-001 renderer→window→conn→pty, pty drop 触发 INV-008 reader→master→child→master_fd。INV-001/002/008/009 全部维持。

- Term<VoidListener> (no-op listener)
- 自建 TermSize impl Dimensions (alacritty 的 TermSize 在 #[cfg(test)] 必须自己做)
- total_lines == screen_lines (Phase 3 T-0304 前无 scrollback)

### [P0-10] advance + cursor_point 喂字节 ✅
手测 `col=17 line=0` 精准匹配 bash prompt `[user@userPC ~]$ ` 长度 17 字符, 证 Processor 正确解析 ANSI/OSC 转义 (OSC 0 title + DECSET 2004 不影响 cursor, 只可见字符落 grid)。
