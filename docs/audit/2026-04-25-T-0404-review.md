### [P0-6] glyph_rasterize_at_2x 测试用 Family::Monospace (而非 primary_face_name) ⚠️ 部分接受 + P3 跟进

**writer 论据**: `Family::Name(&ts.primary_face_name)` 在 closure 内 + Buffer::new(&mut ts.font_system) 同时存在 borrow conflict (closure 同时借 ts.font_system mut + ts.primary_face_name imm)。

**独立 verify (Rust NLL split-borrow)**:

closure 签名 `|ts: &mut TextSystem, font_size: f32| -> Option<RasterizedGlyph>` —— ts 是参数 (不是闭包捕获), 通过 `&mut TextSystem` 参数访问字段时 NLL split-borrow 应支持:
- `&mut ts.font_system` 字段 mut 借
- `&ts.primary_face_name` 字段 imm 借
- 两字段不相交, NLL 允许并存

**但 closure 内的 NLL split-borrow 是 Rust 已知的 edge case** —— 旧 NLL borrow checker 经常在 closure 内退化为"借整个 &mut self" (即不做 field-disjoint split), 实际编译能否通过依 rustc 版本 / 闭包推断细节。writer 选 Family::Monospace 避开此区域, **是合理 simplification**, 不是真假阳性。

**等价替代 (1-line 改写, 避开 closure borrow 限制)**:
```rust
let primary_face_name = ts.primary_face_name.clone();  // 提到 closure 外
let raster_at = |ts: &mut TextSystem, font_size: f32| -> Option<RasterizedGlyph> {
    ...
    let attrs = Attrs::new().family(Family::Name(&primary_face_name));
    ...
};
```

**测试目的判断**:
- 测试名 `glyph_rasterize_at_2x_size_returns_larger_bitmap` —— 验证 "font_size × 2 → bitmap × 2" 的物理关系
- **不是**验证 face_lock (T-0407 已专测 `atlas_key_face_id_matches_primary_for_ascii`)
- Family::Monospace 在用户机仍命中 monospace face (DejaVu Sans Mono / Source Code Pro / Noto Mono fallback), 命中哪个具体 face 不影响 "font_size × 2 → bitmap × 2" 物理关系

**独立判**: ✅ **接受 Family::Monospace 简化路径** —— 测试目的与 face 选择正交, 简化避开 closure borrow edge case。**P3 跟进**: 若 Lead 要求测试也 verify face 锁定, 1-line 改写 (clone primary_face_name 出 closure) 即可, 当前不阻塞。writer 注释 line 962-966 已明确 "Family::Monospace 在用户机仍能命中 monospace face, 比 Family::Name(&primary) 借 self 字段更省事"。
