## P0: unsafe 正确性

### [P0-1] 两处 unsafe 块 ✅

**位置 1**: render.rs line 82-107 — `Renderer::new`
- SAFETY 注释覆盖 3 条约束:指针合法、生命周期 ≥ Renderer、wgpu 不 drop wl 对象
- NonNull::new 验证 null
- 无可能的 panic 破坏不变式

**位置 2**: window.rs line 220-226 — `init_renderer_and_draw`
- display_ptr 来自 `Connection::backend().display_ptr()` 带 null 检查
- surface_ptr 来自 `WlSurface::id().as_ptr()` 带 null 检查

两处都合规。

### [P0-2] Drop 顺序 ✅ 但注释错 🔴(已修)

State 字段声明顺序:registry_state / output_state / **renderer** / window / conn / core

Rust 实际是**正向析构**(按声明顺序,第 1 个先 drop),所以 renderer(3rd)在 window(4th)/conn(5th)之前 drop。**结果正确**。

但代码中三处注释声称 "Rust 反向析构"是**错误的**:
- render.rs:59-60
- window.rs:190
- window.rs:222

若后续维护者读"反向析构"以为要把重要字段放前面,把 renderer 挪到 window/conn 之后,立刻 UB。

**已修**:Lead 在 commit 47dc5f4 改为"按声明顺序正向析构"+ 建 `docs/invariants.md` 的 INV-001/INV-002 登记。

### [P0-3] Renderer 内部 drop 顺序 ✅

`surface → device → queue → config → clear → instance`(正向析构)。surface 先,instance 最后,符合依赖(surface 依赖 Vulkan 实例)。

### [P0-4] panic 路径 ✅

unsafe 块内:NonNull::new 返回 Option 不 panic;create_surface_unsafe 返回 Result 有 .context();pollster::block_on 极少 panic。可接受。

## P0-5: ADR 0001 compliance — forbid→deny ✅

ADR 0001 line 49-51 允许"通过 `// SAFETY:` + 局部 `#[allow(unsafe_code)]` 放行"。T-0102 改:
- lib.rs:`#![forbid(unsafe_code)]` → `#![deny(unsafe_code)]`
- 两处 unsafe 都 `#[allow]` + SAFETY 注释

完全合规。forbid 在 crate 根无法被 inner allow 降级,所以需要降 deny。

## P2: 架构 + 新依赖

### [P2-1] 新依赖合理性 ✅

| 依赖 | 版本 | 理由 | 合理 |
|---|---|---|---|
| wgpu | 0.29 | 技术栈锁 | ✅ |
| pollster | 0.4 | 同步 async 初始化,轻,不拉 tokio | ✅ |
| raw-window-handle | 0.6 | 构造 Raw{Display,Window}Handle | ✅ |
| wayland-backend client_system | 0.3 feature | wgpu Vulkan 需要真 libwayland 指针 | ✅ 必需 |

**wayland-backend client_system feature 必需性**:wayland-backend 有 rust 纯 Rust 和 client_system libwayland FFI 两后端;wgpu Vulkan 要真 libwayland 结构体指针,纯 Rust 后端产出的指针指向 Rust 结构体 ABI 不兼容。论证正确。

### [P2-2] Renderer API ✅

`unsafe fn new(display_ptr, surface_ptr, w, h) -> Result<Self>` + `fn render(&mut self) -> Result<()>`。简洁合理,无 getter(状态不透明,T-0103 自决何时 render)。

### [P2-3] pollster::block_on idiomatic ✅

wgpu 0.29+ async 初始化标准。pollster 是轻量解,符合"不拉 tokio"单线程约束。
