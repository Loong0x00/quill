## P0: 等价性重构验证

### [P0-1] 重构架构总览 ✅

新加三个抽象:
- **WindowCore (pub)** — 四个核心字段 + 新增 `resize_dirty` 标志
- **WindowEvent (pub)** — 事件枚举:Configure/Close/Disconnect
- **handle_event (pub)** — 单步状态转移纯逻辑

设计合理:纯逻辑与 Wayland FFI 分离,便于 headless 测试;state machine 经典签名 `fn(&mut WindowCore, WindowEvent) -> WindowAction`。

### [P0-2] 行为改动 1:0x0 防守 🟡 scope 内

**原版**(main line 238-252):若 compositor 发 Some(0),会使用 0 → width/height 可能被设 0

**新版**:若任一轴为 0,整条 configure 吞掉,保留旧尺寸
```rust
let w = new_width.unwrap_or(state.width);
let h = new_height.unwrap_or(state.height);
if w == 0 || h == 0 {
    return action;
}
```

xdg-shell 协议 0 表示"client 决定",defensive 拒绝是对的。task notes 要求"0x0 被忽略"测试,改动符合。

### [P0-3] 行为改动 2:后续 configure 尺寸更新 🟡 scope 内

**原版**:后续 configure 被忽视(注释"避免 SlotPool 容量越界"),width/height 不变

**新版**:后续 configure 更新 width/height,但只在 first_configure 时画占位
```rust
} else if w != state.width || h != state.height {
    state.width = w;
    state.height = h;
    state.resize_dirty = true;
    action.needs_draw = true;
}
```

draw_placeholder 仍然只在 was_first 时执行(line 342),保护了原版"避免 SlotPool 容量越界"意图。为 T-0103 (wgpu swapchain) 铺路。

### [P0-4] 核心状态转移对等性 ✅

| 场景 | 原版 | 新版 | 等价 |
|---|---|---|---|
| 首次 configure(w×h) | 设 width/height, first_configure=false | 同 + resize_dirty=true | ✅ |
| 首次 configure(0×0) | width=0/height=0 | 吞掉保留旧值 | ⚠️ 改进 |
| 后续 configure 尺寸变 | 忽视 | 更新 + resize_dirty | ⚠️ 改进 |
| 后续 configure 尺寸同 | 忽视 | 不置 resize_dirty | ✅ |
| Close | exit=true | exit=true (via handle_event) | ✅ |

### [P0-5] resize_dirty 字段 ✅

置位:首次/后续 configure 尺寸变化。清零:上层(T-0103 wgpu)消费后。语义:布尔(连续 resize 合并到单次脏标记)。

测试覆盖:`idempotent_same_size_configure_does_not_re_dirty` + `consecutive_resizes_merge_to_single_dirty`。

### [P0-6] 新测试套质量 ✅✅

9 个测试(实际 10),全 unit test 无 I/O。覆盖 acceptance 全 case:

| Acceptance | 对应测试 |
|---|---|
| 初始 configure | `initial_configure_applies_size_and_clears_first_flag` + `initial_configure_with_none_falls_back_to_initial_dims` |
| 后续 resize | `subsequent_resize_updates_dims_and_sets_dirty` + `idempotent_same_size_configure_does_not_re_dirty` |
| 0x0 被吞 | `zero_size_configure_is_swallowed` + `zero_size_on_first_configure_does_not_flip_first_flag` |
| close 置 exit | `close_sets_exit_flag` |
| disconnect 视同 close | `disconnect_sets_exit_flag` |
| 连续 resize 合并 | `consecutive_resizes_merge_to_single_dirty` |
| 端到端 | `full_lifecycle_init_configure_resize_close` |
| 部分尺寸 | `partial_size_configure_keeps_other_axis` |

命名清晰、AAA 结构、`fresh_core()` 工具函数减少重复。无 mock 陷阱、无测试自己。
