# ADR-0020: kernel 升为开机无头服务 + 无锁会合(会合永不阻塞终端;终端永远优先)

状态: Accepted(设计定稿 2026-07-02,**未实现**;修订/延伸 ADR-0018 E′ 与 ADR-0019 联邦。也是修联邦 #3 卡顿的 spec)
关联: ADR-0018(E′ 共享=隔离子进程,终端绝对优先)、ADR-0019(多窗口/多设备联邦 + 共享锚窗口=home)、CLAUDE.md

## Context

ADR-0019 落地(feat/federation)后,对抗审逮到 **#3:开/加入共享会同步阻塞 calloop 单线程最长 ~3s(flock + 轮询 connect),期间终端冻结**。顺着 #3 往下,用户驱动逼出两个更深的结论:

1. **会合的锁在防一个【实际产生不出】的竞态,却赔上了 #3 这个真回归**。竞态 = "两个窗口几乎同时开共享 → 都检查'无 kernel' → 都 spawn"。但:① 共享是**手动 opt-in**(`--share` / 标题栏开关),正常启动是不带 `--share` 的 `quill`;② **人手做不到**在第一个 kernel 绑好前(亚秒)去第二个窗口再开共享(鼠标/alt-tab 都远超那个窗口);③ 现实里没有脚本会"专门 `quill --share`、还开两个、还是交互终端";④ 就算真撞(未来"登录恢复窗口+共享态"之类),**输的 kernel bind 失败秒退即自愈**。→ 为不可达威胁上锁 = 过度设计 + 引入真卡顿。

2. **开机自启动要能"没人坐在终端前就 serve" → 那个自启的东西必须是【无头服务】,不能是 GUI quill**。GUI quill 要 Wayland 会话/显示器/你已图形登录才成立;而远程访问的意义正是"你还没到机器前就要够回来"。所以 kernel 该是一等的开机服务,GUI 窗口是可选的连接端。

## Decision

### 1. kernel = 开机无头服务(可独立于任何 GUI 运行)

- kernel 可作为 **systemd user service** 开机自启:**无需 GUI / 无需人在场即 serve**(bind 7878 + 会合 socket)。
- GUI quill 窗口降为**可选的 feeder / viewer**:在场则 attach、不在场服务照跑。
- kernel 因此能拥有"**远程起的、没有本地窗口**"的会话(你人不在,手机上开个 shell,它活在服务里;回家某个窗口再看)。
- 与 E′ 不冲突:**本地窗口仍 own 自己 PTY、直接渲染(热路径零 IPC、快)**;kernel 只额外拥有"远程无本地窗口"那类会话 + 做中继。两种 PTY 归属并存(窗口拥有=本地 E′ 快 / kernel 拥有=远程会话)。**不做 session-server**(本地窗口渲染绝不过 IPC,ADR-0018 已否)。

### 2. 无锁会合(lock-free),连接异步(永不阻塞终端)

- **删掉 flock + 轮询整套**。会合退化成**纯顺序**逻辑:
  - **先 try-connect** 会合 socket:连上(kernel 已在跑,可能是开机服务、也可能别的窗口拉起的)→ 作为 feeder join。
  - 连不上(没 kernel)→ spawn kernel(或它本就该由 systemd 起)。
- **无竞态要防**(见 Context 1),故**不需要任何锁 / 原子仲裁**。
- **万一**未来出现真并发双 spawn:**靠 kernel bind 7878 的原子性兜底** —— OS 保证只有一个 bind 成功,输的秒退。这是**普通错误处理**(接现有"启动失败→标题栏琥珀"提示),**不是防竞态的机器**。
- **连接全程 calloop 异步、非阻塞**:发起 connect 立即返回,"连上没 / kernel 起来没"交给事件循环**每圈顺手查**(retry 用 calloop timer,非 `thread::sleep`)。**任何一步都不在循环线程上死等** → 终端一秒不冻。**#3 由此消失。**

### 3. ★ 终端永远优先(重申,E′ 的立命之本,载重约束)

**quill 首先是一台终端。** 一个 GUI 窗口的启动/运行次序与优先级**恒为**:

1. **先起终端 + 开一个 tab**(own PTY、直接渲染、E′、完全独立、快);
2. **然后**(作为**额外、之后、非阻塞**的一步)连接服务当 feeder。

- **连服务是叠加在终端之上的尽力而为的一层**,**绝不阻塞、绝不拖累终端**。
- 服务**挂 / 慢 / 不在 / 连不上 → 终端毫不受影响**,照常是今天的 quill。
- 这条统摄前两点:kernel 变服务、会合变无锁异步,**都不得以任何形式让"终端本身"退化**。#3 就是违反了它(会合把终端冻了),本 ADR 的无锁+异步正是为把这条焊死。

## 被否决的方案

- **自启动 GUI quill**:要显示器/会话/人在场 → 与"没人时也能远程够到"自相矛盾。→ 自启的必须是无头服务。
- **flock 会合(ADR-0019 原样)**:防不可达竞态、且同步阻塞引入 #3。→ 无锁 + 异步取代。
- **session-server(kernel own 所有 PTY、窗口=IPC 视图)**:本地渲染回到热路径 IPC,ADR-0018 已否。→ 本地窗口仍 own PTY 直渲。

## Consequences

- ✅ **开机即可远程够到,无需人在场**(kernel systemd service)。
- ✅ **#3 消失**(无锁 + 异步会合,终端零阻塞)。
- ✅ 代码更简单(删 flock + 轮询一整套)。
- ✅ 新能力:kernel 拥有的"远程无本地窗口"会话。
- ⚠️ 需写 systemd user service 单元;kernel 要能**独立(零 feeder)启动并运行**(不再假设"总有一个发起窗口")。
- ⚠️ 两种 PTY 归属并存(窗口拥有 / kernel 拥有),生命周期各自清晰:窗口拥有随窗口、kernel 拥有随服务 + refcount。
- ⚠️ 本地窗口↔服务的 attach 是**可选、异步、可失败**的:必须保证 attach 失败/服务缺席时终端完全正常(§3)。

## 与在飞 feat/federation 的关系(重要)

feat/federation(ADR-0019 F1+F2+F3)已实现"带 flock 的同步会合",正是 #3 的来源。**修 #3 = 按本 ADR 把会合改成【无锁 + try-connect-else-spawn + 连接异步】**,并保证 §3 终端优先。kernel 的"可作 systemd 服务独立运行 / 拥有远程会话"可作后续砖(先把会合无锁异步 + 终端优先做到,服务化紧随)。

## 分砖(建议)

- **砖 G1(= 修联邦 #3)**:会合改无锁(删 flock)+ try-connect-else-spawn + 连接走 calloop 异步(非阻塞);bind 失败接现有琥珀提示。**先合这个**(联邦才可用、终端不冻)。
- **砖 G2**:kernel 能独立(零 feeder)启动运行 + systemd user service 单元 + 开机自启。
- **砖 G3**:kernel 拥有的"远程无本地窗口"会话(远程新建落 kernel、GUI 窗口可后 attach 观看)。

## 非目标 / 推迟

- G2/G3(服务化 + 远程会话)是后续;**先做 G1**(无锁异步会合)把 #3 解决、联邦可合。
- 多设备(ADR-0019 F5)不变,仍待 Mac 无头移植。
